Description

What the suite is and what it tests.

This competition challenges researchers to evaluate their global optimization algorithms on a carefully crafted set of 24 problem instances generated using the Generalized Numerical Benchmark Generator (GNBG) [1]. The suite spans a diverse range of optimization landscapes, from smooth unimodal surfaces to rugged multi-component multimodal terrains. The newly designed suite follows the same baseline function as used in GECCO 2024 competitions, with modified instances to add further complexity.

Suite composition
  • Unimodal instances (f1 to f6)
  • Single-component multimodal (f7 to f15)
  • Multi-component multimodal (f16 to f24)

Rules and Details

Participation rules and constraints.

  • The competition is open to all researchers and practitioners in continuous numerical optimization.
  • Competitors can participate with either new (unpublished) or previously published algorithms.
  • Winners and runners-up will receive certificates from the conference.
  • It is not required to attend the conference or register in order to participate in the competition.
  • No modifications are permitted to parameter settings of the instances in the “.mat” files.
  • Algorithm parameter values must be consistent across all instances. Per-instance tuning is prohibited.
  • Instances must be treated as black boxes. Direct use of GNBG internal parameters is forbidden.
  • Winners will be required to share source code for verification (kept confidential).

Evaluation Criteria

Computed over 31 independent runs per instance.

1 Average Absolute Error

Mean absolute error of best-found solutions across 31 runs (accuracy and consistency).

2 Average FEs to Threshold

Mean function evaluations to reach absolute error < 10-8 (efficiency).

3 Success Rate

Percentage of runs achieving absolute error < 10-8 (robustness).

Stop criterion: maximum function evaluation number specified per instance. For f1 to f15, max FEs = 500,000; for f16 to f24, max FEs = 1,000,000.

Submission Instructions

What to submit and how to format it.

Submit a compressed folder labeled with the name of your algorithm containing:

Documentation file
  • Title of the submission
  • Names, affiliations, and email addresses of all team members
  • A concise description of the algorithm
  • A table presenting mean and standard deviation across 31 runs for each instance:
    • Absolute error
    • Required FEs to reach acceptance threshold
    • Success rate
